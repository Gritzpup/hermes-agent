- **Key ports**: `:4300` (API), `:4395` (bridge), `:16380` (Redis), `:18789` (openclaw gateway)
- **Tilt**: Runs as a **systemd service** on port 10350 — always alive. Commands (`tilt get uiresource`, `tilt up`, `tilt down`, `tilt edit`) talk to the daemon. Use `tilt get uiresource <name> -o json` for full JSON status. Check logs via `tilt logs <resource>`. **NEVER restart the systemd service** — Tilt auto-reloads Tiltfile changes. Use `tilt down <resource>` then `tilt up <resource>` to restart individual resources.
- **COO endpoints**: `POST /api/coo/pause-strategy`, `/api/coo/amplify-strategy`, `/api/coo/directive`

## Conventions

- TypeScript: `npm run check --workspace @hermes/api` before committing
- Never push to `origin/master` directly — `coo-journal-committer` auto-pushes every 6 h
- Never write `/api/coo/*` routes without per-route `express.json()` middleware
- `openclaw-agent-cleanup`: grep pattern must be `"openclaw +agent( |$)"` not bare `"openclaw-agent"`
- **Tilt-first**: Always use `local_resource` in Tiltfile for services. Docker only for third-party images (postgres, redis). Restart servers via `tilt down <resource>` then `tilt up <resource>`. View logs with `tilt logs <resource>`. **NEVER run `systemctl restart tilt`** — the daemon auto-reloads Tiltfile changes. Kill/pkill/node only as last resort.
- Strategy-director P&L attribution is known to be **broken** — always cross-reference against `/api/coo/directives` journal source
